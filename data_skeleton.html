Skeleton Raw data

There are 36 group, 108 experiments across three different tasks and communication media conditions in our study. Hence, we collected 216 motion log files in our dataset.

We used Microsoft Kinect to track users' body movement in the experiment and recorded it as skeleton data in 30 frames per second. We then formatted the skeleton data captured by Microsoft Kinect into plaintext. Researchers who want to analyze the data for further usage can read the data by a formatter script and transform it into any kind of format.



Data Structure

  • File Format: TXT, each TXT file contains skeleton data per single participant in each session.
  • File Counts: 216(18000 frames per file)
  • Groups: 36 (EL1-EL2:18,EL2:EL2:18)
  • Participants per Group: 2, total 72 participants.
  • Media Conditions: Audio, HandVis, Face-to-Face

File Format

Example image shows the format of the skeleton data we got from the experiment. We separated each skeleton session by individual points in time; each time point occurs once every 1/30 seconds, according to the frame speed of Microsoft Kinect. In each skeleton session, we log skeleton joints’ position of the user. Each joint element denotes a position of the user’s skeleton in the 3D space which contains individual’s three-dimensional positions (with x, y, z values) of 10 skeleton joints (head, shoulder, elbow, wrist, hand) at the time point. The length unit of each three-dimensional position is meters, we recorded them up to two decimal places.



Equipment

In our experiment, We put two Kinect cameras in the middle, directed outward toward each participant to capture their skeleton data without showing RGB streaming video and system interface in front of them in other two modes.

Kinect Taping toolkit was developed for sensing and recording users’ skeleton data and then transforming it into visualization effects with Microsoft Kinect Standard Development Kit (SDK) v1.8. With Kinect Sensor, users’ skeleton data such as head, hand position and face feature points, can be captured at the rate of 30 frames per second. Each joint data point contains a 3D coordinates position (x, y, z), and we recorded it with timestamp for each 1/30 seconds and positional values (x, y, z) of each joint at that time.

Moreover, in the experiment, participants could only sit on a chair which was at a 120cm distance from the Kinect sensor and monitor. Before the start of each experiment, they were told to sit still and not to stand up, in order to prevent periods of lost tracking by the Kinect.
